process¶

process.process Module¶

The process module contains two classes that are used to process the data. The first class ProcessCCD is used for active CCD imagers (ACS, STIS, and WFC3). Each of the active CCD imagers has a pythonic interface to their calibration pipelines, which includes their cosmic ray rejection routines listed below:

  • ACS

    • acstools.acsrej

  • STIS

    • stistools.occreject

  • WFC3

    • wfc3tools.wf3rej

The second class ProcessIR is used for both active and inactive IR imagers. The IR imagers used on HST utitilize the method of “up-the-ramp” sampling to identify cosmic rays which are subsequently marked in the data quality (DQ) arrays of the IMA files. For more information, see Chapter 3.3.10 in the WFC3 DataHandbook. Since IMA files already have the cosmic rays identified, the function of the ProcessIR is to decompose the IMA into a series of individual FITS files corresponding to each read.
