pipeline¶

usage: python pipeline.py [-h] [-aws] [-download] [-process] [-ccd] [-ir]
                          [-chunks CHUNKS] [-analyze] [-use_dq] [-instr INSTR]
                          [-initialize]

Named Arguments¶

-aws

Flag for using AWS for downloads. Only to be used when the pipeline is run on EC2

Default: False

-download

Download the data

Default: False

-process

Process the raw data

Default: False

-ccd

Switch for processing CCD data

Default: False

-ir

Switch for processing IR data

Default: False

-chunks

Number of chunks to break the generated results into. For example, if -chunks 2 is passed, then two HDF5 files for each statistic will be generated. The first half of the dataset will be written to file 1 and the second half will be written to file 2. This is to offset the degradation in write time as the number of datasets stored in the HDF5 increases.

Default: 4

-analyze

Switch for analyzing and extract cosmic ray statistics

Default: False

-use_dq

Switch for using the DQ arrays to perform labeling

Default: False

-instr

HST instrument to process (acs_wfc, wfc3_uvis, stis_ccd, acs_hrc)

Default: “stis_ccd”

-initialize

Initialize the HDF5 files for the instrument.

Warning: Should only be included the first time the pipeline is run becuase it will overwrite any pre-existing HDF5 files.

Default: False
